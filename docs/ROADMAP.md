# FridgeTube 차후 개발 로드맵

## 현재: Phase 1 (MVP)
**목표:** $0/월로 핵심 기능 웹+PWA 출시

- ✅ 재료/요리명 통합 검색
- ✅ YouTube 채널 인덱싱 기반 로컬 검색
- ✅ Gemini AI 레시피 추출 + 분량 변환
- ✅ 재료 GAP 분석 (5단계)
- ✅ 사진 재료 인식
- ✅ 선호 유튜버 관리
- ✅ 외부 레시피 텍스트 입력
- ✅ PWA (설치 가능, 오프라인 폴백)

---

## Phase 2: 사용자 + 데이터 확장 (MVP 출시 후 1~3개월)

### 2.1 사용자 인증
- Google/Kakao OAuth 소셜 로그인
- 세션 기반 → JWT 토큰 전환
- 비회원 데이터 → 회원 마이그레이션

### 2.2 레시피 저장/히스토리
- 내 레시피 라이브러리
- 검색 히스토리 기반 추천
- 레시피 메모/수정

### 2.3 데이터 소스 확장
- 레시피북 사진 OCR → 구조화
- URL 입력 → 웹 레시피 스크래핑 (만개의레시피, 네이버 블로그)
- 대체 재료 AI 추천

### 2.4 검색 품질 강화
- 채널 자동 인덱싱 확장 (사용자 추가 채널)
- 검색→클릭→추출 성공률 로깅 → 채널 리밸런싱
- 요리명 DB 운영 로그 기반 주간 확장

### 2.5 비용 최적화
- Gemini 정확도 미달 시 Claude Haiku 전환 ($15~30/월)
- YouTube 쿼터 증액 신청 (컴플라이언스 감사)
- 자체 ASR (Whisper) 검토 — 비공식 자막 대체

## Phase 3: 수익화 + 앱 (출시 후 3~6개월)

### 3.1 프리미엄 구독
- ₩2,900/월: 무제한 레시피 변환, 광고 제거, 사진 인식 무제한
- 무료: 하루 3회 레시피 변환, 영상 추천 무제한

### 3.2 제휴 마케팅
- 부족 재료 → 쿠팡/마켓컬리 장보기 연동 (CPA/CPS)
- 식품 브랜드 네이티브 광고 ("이 레시피에 OO소스 추천")

### 3.3 고급 기능
- 레시피 비교 분석 (같은 요리 다른 유튜버)
- 맛 프로필 시스템 (매운맛 선호 등)
- 유통기한 관리 + 알림

### 3.4 앱 전환 전략
```
Step 1: PWA 유지 + 홈 화면 설치 유도
  └── 데이터: 주간 활성 사용자 중 설치 비율 추적

Step 2: PWA → 앱 스토어 패키징 (PWABuilder)
  └── 조건: 설치 사용자 > 전체의 30%
  └── Google Play: TWA (Trusted Web Activity) 래핑
  └── Apple: PWABuilder 또는 Capacitor 래핑

Step 3: 네이티브 앱 (필요 시)
  └── 조건: 바코드 스캔, 위젯, 백그라운드 알림이 핵심 가치가 될 때
  └── React Native 또는 Flutter
  └── 웹/앱 코드 공유 비율 최대화
```

### 3.5 유튜버 채널 제휴
- 프리미엄 채널 등록 (우선 노출 수수료)
- 채널별 레시피 자동 정리 서비스
