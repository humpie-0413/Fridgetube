# 화면 7개 + PWA

## S01 홈/검색 (/)
- 큰 검색 입력 (placeholder: "냉장고에 뭐가 있나요? 또는 먹고 싶은 요리를 검색하세요")
- 입력된 재료 태그 (x 삭제)
- 자동완성 드롭다운 (GET /v1/ingredients/search)
- 인원수 슬라이더 (1~10, 기본 2)
- 모드 토글 ("영상 추천" / "레시피 변환")
- 검색 버튼 → /results
- 사진 인식 버튼 → S05 모달
- 텍스트 입력 버튼 → S06 모달
- 상태: 초기 | 입력중 | 채널필터ON

## S02 검색 결과 (/results)
- 2탭 구조:
  - [탭1: 내 채널 추천] 로컬 인덱스 결과, GAP 뱃지 가능, 자유 정렬
  - [탭2: YouTube 검색] search.list 결과, 원본 순서, 뱃지/재정렬 금지
- 영상 카드: 썸네일 + 제목 + 채널 + 조회수 + "~N개 부족" 뱃지
- 정렬: 인기순 / 예상 부족 적은 순 (탭1만)
- 상태: 로딩(스켈레톤6개) | 결과있음 | 빈상태 | 오류

## S03 레시피 상세 (/recipe/[id])
- 상단: YouTube 영상 임베드
- 2탭: 원본 레시피 / 변환 레시피
- 인원수 슬라이더 (실시간 재계산)
- 재료 체크리스트 (GAP 상태 색상 구분)
- 조리 순서 스텝
- 장보기 목록 (부족 재료만)
- base_servings_source가 inferred/default면 "인분 추정치" 배너
- 상태: 로딩(AI분석중 프로그레스) | 완료 | 추출실패 | 낮은신뢰도

## S04 채널 관리 (/channels)
- 채널 검색 입력
- 검색 결과 카드 (아바타 + 이름 + 구독자 + 추가 버튼)
- 내 채널 목록 (아바타 + 이름 + 삭제 버튼)
- 상태: 로딩 | 결과 | 빈상태("좋아하는 유튜버를 추가하세요")

## S05 사진 인식 (모달)
- 카메라/갤러리 버튼
- 업로드 프로그레스
- 인식 결과 태그 (수정/삭제 가능)
- 각 태그: 유사 후보 드롭다운 ("대파" → "쪽파/부추/실파")
- "기본 양념 체크" 섹션 (소금/간장/식용유 등 토글)
- 확인 → S01 태그로 전달
- 상태: 초기 | 업로드중 | 분석중 | 결과 | 인식실패

## S06 외부 레시피 입력 (모달)
- 텍스트 영역 (placeholder: "레시피를 붙여넣으세요...")
- 구조화 버튼 → API 호출
- 결과: 레시피 카드 미리보기
- 확인 → S03으로 이동
- 상태: 초기 | 분석중 | 결과 | 실패

## PWA
- manifest.json: name="FridgeTube", short_name="냉장고튜브", display="standalone", theme_color="#16a34a"
- 앱 아이콘: 192x192, 512x512
- 서비스 워커: 오프라인 폴백 페이지 ("/offline")
- 설치 프롬프트: beforeinstallprompt 이벤트 처리
- meta: viewport, theme-color, apple-touch-icon

## 공통 UI 패턴
- 로딩: 스켈레톤 UI (shimmer)
- 빈 상태: 일러스트(SVG) + 안내 메시지
- 에러: "문제가 발생했어요" + 재시도 버튼
- 토스트: 성공/에러 알림 (하단)
- 모바일: 360px~ 반응형
- 색상: primary=#16a34a(green-600), 배경=#fafaf9
